Intercept Feature

This page registers a service worker that caches requests according to the rules below. Open DevTools → Application → Service Workers to see it active. 

intercept /
    precache /, /test/manual/offline.html as "hs-intercept-v1"
    on /api/* use network-first
    on *.css, *.js use cache-first
    on /offline-target use network-only
    offline fallback /test/manual/offline.html
end

1. API request (network-first)

Fetches /api/hello. Each request returns a fresh timestamp from the server. Network-first means: try network, fall back to cache. Turn off your network in DevTools and click again - you should get the previous cached response.

4. Offline fallback

Turn on "Offline" mode in DevTools Network tab, then click the button below. The request goes to an unmatched path that doesn't exist in cache - the service worker falls back to /test/manual/offline.html instead of failing.
